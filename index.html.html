❓ Help & Setup Guide

🚀 Quick Start (3 Steps)
  1. Switch to the 🔗 Trend: Pipeline tab
  2. Paste your ADO Personal Access Token → click Fetch Builds
  3. Select a POD from the dropdown → click Analyze Trend
That's it — you'll see trend data, classifications, and triage tools for all tests in that POD.
🔑 PAT Setup
Your ADO Personal Access Token needs these scopes (select Custom defined when creating):
ScopePermissionUsed For
BuildReadFetch pipeline builds & download artifacts (ZIPs)
CodeRead & WriteRepository access for code-level operations
Work ItemsRead, Write & ManageBug creation, US title & assignee lookup, work item management
How to create a PAT:
  1. Go to Azure DevOps → click your profile icon (top-right) → Personal Access Tokens
  2. Click + New Token
  3. Name it (e.g. "Allure Viewer"), set Organization to your org, set Expiration (max 1 year)
  4. Select Custom defined under Scopes, then check the 3 permissions above
  5. Click Create → copy the token immediately (you won't see it again)
The PAT is saved in your browser's localStorage and never sent anywhere except ADO APIs over HTTPS.
📑 Tabs
📄 Single Report — Drop or browse a local Allure HTML file. Parses it instantly and shows all tests with grouping, filtering, triage, and bug creation.
🔗 Trend: Pipeline — Connects to Azure DevOps, downloads artifacts from recent pipeline builds, and compares test results across builds for trend analysis.
📊 Understanding the Dashboard
Classification Cards
● Consistent Failure — fails in every build
● Flaky — alternates pass/fail
● Newly Broken — was passing, now failing
● Recently Fixed — was failing, now passing
● Stable — passes consistently
Click cards to filter the table.
Triage Workflow
1. Click the ● dot next to a test
2. Set status: Known Issue, Fixed, Bug, or Blocked
3. Add notes and optionally link a Work Item ID
4. Progress bar tracks how many failures are triaged
Triage data persists in localStorage across sessions.
🪟 Split Mode
Check Enable Split Mode (or press s) to get a master-detail view. The left pane shows the test list, the right pane shows full details for the selected test — including steps, screenshots, failure traces, similar tests, and triage controls. Use ↑↓ to navigate.
🛠️ Toolbar Buttons
🔥 Domain Heatmap — pass rates per domain across builds (requires Analyze Trend first)
🌐 Global POD Heatmap — pass rates for ALL PODs across all builds (available after Fetch Builds)
📈 Test Growth — weekly test count trend with pass/non-pass stacked bars
🗺️ Coverage Map — Miller-column drill-down: Domain → User Story → Test
🔍 Analyse — per-test build-by-build breakdown with screenshots and traces
🐛 Create Bug — copies repro steps + screenshots to clipboard, opens ADO pre-filled
📁 Disk Cache
Click Set Cache Folder to pick a local directory. The tool stores downloaded ZIP artifacts there so subsequent loads skip the download. It also caches User Story titles for the Group By → User Story view. Works in Chrome and Edge only (File System Access API).
☑️ Bulk Actions
Use checkboxes to select multiple tests. A bottom bar appears with options to: set triage status in bulk, add shared notes, or create a single bug covering all selected failures.
⌨ Keyboard Shortcuts
Press ? or click the ⌨ button to see all shortcuts. Key ones: s split mode, / search, h heatmap, t theme, e expand/collapse all.
